## 为什么
### 对于程序猿来说
* HTTP 只是表面上简单
* 底层的原理非常复杂
* 很少有人去深究 HTTP 背后的细节
* 现在流行的 REST 架构设计

### 对于运维喵
* 部署环境应用
* 应用 HTTPS 安全
* 大型分布式设计

这些都离不开 HTTP 底层原理

但是 
* 网上资料太杂不成体系
* 我们往往知其然而不知其所以然
* 甚至还有不争取的低档

### 测试媛
* 在网上找课程，又都过于表面
* 最重要的还是过于枯燥

## 互联网必备的 HTTP 协议基础
* HTTP 工作方式
* HTTP 协议结构
* HTTP 通讯原理
* HTTP 协议特性

往大了说，影响着互联网的进步和未来
往小了说，很多设计工作也是离不开这些知识的掌握
